## 零：配置环境
### 1. 安装所需的依赖包
- 在项目文件夹下打开PowerShell
- 查看一下Python是否安装，版本是否符合要求 `python --version`
- 先`python -m venv .venv`后`.\.venv\Scripts\activate`，创建虚拟环境并激活(好处就是可以把不同项目的依赖项环境隔离开，防止不同项目之间的依赖项版本之间发生冲突。)
- 安装所有项目依赖 `pip install -r requirements.txt`
- `pip list` 查看该有的依赖是否都安装成功

### 2. 填写好 .env 文件里的信息
- 要从 Notion 自建的数据表里提取数据，需要以下信息：Notion Token，数据表对应的 ID（Database ID），用于定位对应内容的变量（例如本项目中是 date，通过 date 拿到对应的 text）。
- 要通过 Ollama 调用大模型，就需要本地的服务端口号。一般 Ollama 在安装的时候，都默认了一个本地运行的服务端口，也就是`http://localhost:11434`
- 要将外部信息发到飞书里的群聊中，就需要通过 Webhook 这个端口链接。

# 注意事项
### 1. 不同项目通过创建venv虚拟环境将各项目的依赖项独立开
- `python -m venv .venv`
- `.\.venv\Scripts\activate`
- `pip install -r requirements.txt`
### 2. 每次要跑代码的时候，记得用以下代码先进入vev环境
- `.\.venv\Scripts\Activate.ps1` 成功进入 venv 环境之后，终端会显示 venv 的标识。由于项目之间的依赖项是隔开的，分别打包在各自的 venv 环境里，所以如果要复现项目的执行结果，就需要进入对应项目的 venv 环境里，再去跑执行代码才可以。
### 3. 版本管理
- 把能跑通的 MVP 先上传到 GitHub，这样保证有一个最小可行性的版本，后面再做新功能添加的时候就更容易做版本控制。
- 上传到 GitHub 的项目中，可以除去以下内容，保持代码的简洁：
1. venv 虚拟环境文件。
2. .env文件，里面包含个人的隐私密钥。
3. 缓存临时文件：如 __pycache__。
4. 本地测试产物：如 .log、.png 等文件。
