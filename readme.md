## 零：配置环境
### 1. 安装所需的依赖包
- 在项目文件夹下打开PowerShell
- 查看一下Python是否安装，版本是否符合要求 `python --version`
- 先`python -m venv .venv`后`.\.venv\Scripts\activate`，创建虚拟环境并激活(好处就是可以把不同项目的依赖项环境隔离开，防止不同项目之间的依赖项版本之间发生冲突。)
- 安装所有项目依赖 `pip install -r requirements.txt`
- `pip list` 查看该有的依赖是否都安装成功

### 2. 填写好 .env 文件里的信息
- 要从 Notion 自建的数据表里提取数据，需要以下信息：Notion Token，数据表对应的 ID（Database ID），用于定位对应内容的变量（例如本项目中是 date，通过 date 拿到对应的 text）。
- 要通过 Ollama 调用大模型，就需要本地的服务端口号。一般 Ollama 在安装的时候，都默认了一个本地运行的服务端口，也就是`http://localhost:11434`
- 要将外部信息发到飞书里的群聊中，就需要通过 Webhook 这个端口链接。

## 二、Python调用Ollama里的模型读取文本里的时间信息
### 1. 下载安装配置oLLama，下载开源模型Qwen3：4b模型,并验证模型是否能够成功运行
- 任何一个目录下打开powershell输入`ollama pull qwen3:4b` 下载模型
- `ollama run qwen3:4b` 和大模型对话，看能不能得到回复，能的话证明大模型能成功运行
### 2. 代码验证
- `python llm_time_extract.py 2026-07-02` 终端如果能够打印出JSON形式的信息，说明这一步跑通了

# 项目已更新部分
### Version 1.1 版本更新

- [x] 自动读取notion日程页面的某一天的信息，根据日期自动填入raw time logs这个数据库里

### Version 1.2 版本更新
- [x] 总共合并成一个数据库（将日程活动合并到数据库的第一列的页面里了，但同时仍然保留了日程记录页面）
- [x] 需要写一个脚本，将原本按照折叠标题记录的日程，给批量转移到数据库里去
- 这个版本重构了正工作流的链路。首先，我打算仍然保留日程页面，这也是我每天记录日程活动的页面；然后，通过 migrate 这个代码文件，会把这个页面对应日期下面的内容，写到数据库里的一行里面去。
- `python migrate_schedule.py --start 2026-07-01 --end 2026-07-08 --overwrite-body --with-ai --sleep-hours 8` 这就是标准的使用，跑起来这个版本的格式。其中睡觉时间`--sleep-hours 8`不写的话，默认就是7.

### Version 1.21 版本更新
- 在版本 1.2 的基础上，对代码的函数进行了一些归类以及整理，将功能相似的函数放在同一个代码文件里，使得代码结构更加清晰。
- 支持单一一天的处理，也支持一个时间段的批量处理。`python migrate_and_summary.py --date 2026-07-08` 可以处理单日。
`python migrate_and_summary.py --start 2026-07-01 --end 2026-07-08 --overwrite-body --clear-snapshot --clear-summary` 可以批量按时间顺序处理一个时间段
`python migrate_and_summary.py --start 2026-07-01 --end 2026-07-08 --no-feishu` 只批量写Notion而不发飞书
